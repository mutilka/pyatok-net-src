Чтобы добавить отдельную папку с изображениями для страницы https://gorshochek.by/zakuski/178-sukhariki-k-pivu-poshagovyj-retsept-s-foto в Hugo, можно воспользоваться механизмом page bundles (пакетов страниц). Это позволит организовать контент и связанные с ним ресурсы (например, изображения) в одной директории. Давай разберём, как это сделать.
Использование Page Bundle
В Hugo есть два типа пакетов страниц: Leaf Bundle (для одиночных страниц) и Branch Bundle (для разделов). Для твоей задачи нужен Leaf Bundle, так как это конкретная статья.
Шаг 1: Измени структуру
Вместо файла 178-sukhariki-k-pivu-poshagovyj-retsept-s-foto.md в папке content/zakuski/, создай папку с этим именем, а внутри неё размести файл index.md и папку с изображениями. Вот как это будет выглядеть:
content/
  zakuski/
    178-sukhariki-k-pivu-poshagovyj-retsept-s-foto/
      index.md
      images/
        photo1.jpg
        photo2.jpg
index.md — это основной файл страницы, который содержит фронт-маттер и текст.
images/ — папка с изображениями, которые относятся только к этой странице.
Содержимое index.md:
```yaml
title: "Сухарики к пиву: пошаговый рецепт с фото"
slug: "178-sukhariki-k-pivu-poshagovyj-retsept-s-foto"
date: 2025-03-22
Текст статьи...
```

#### Шаг 2: Работа с изображениями в шаблоне
Изображения из папки `images/` автоматически становятся доступны как ресурсы страницы. В шаблоне или контенте их можно использовать через функцию `.Resources` или прямые ссылки.

Пример в `index.md`:
```markdown
![Сухарики шаг 1](images/photo1.jpg)
![Сухарики шаг 2](images/photo2.jpg)
Или в шаблоне (например, layouts/_default/single.html):
html
{{ range .Resources.ByType "image" }}
  <img src="{{ .RelPermalink }}" alt="{{ .Name }}">
{{ end }}
Hugo сам обработает изображения (например, сжатие, ресайз), если ты используешь встроенные методы вроде .Resize:
html
{{ $image := .Resources.GetMatch "images/photo1.jpg" }}
<img src="{{ ($image.Resize "600x q85").RelPermalink }}" alt="Сухарики">
Шаг 3: URL остаётся прежним
Благодаря структуре Leaf Bundle, URL страницы всё ещё будет /zakuski/178-sukhariki-k-pivu-poshagovyj-retsept-s-foto, как ты хотел. Папка images/ не влияет на путь, так как она является частью ресурсов страницы, а не отдельным URL.
Итоговая структура
content/
  zakuski/
    178-sukhariki-k-pivu-poshagovyj-retsept-s-foto/
      index.md
      images/
        photo1.jpg
        photo2.jpg
Преимущества
Изображения привязаны к конкретной странице и не смешиваются с другими.
Легко управлять: всё, что связано с "Сухариками к пиву", лежит в одной папке.
Hugo автоматически распознаёт ресурсы в таком формате.
Дополнительно
Если ты хочешь общую папку для всех изображений сайта (например, static/images/), это тоже возможно, но тогда изображения не будут привязаны к конкретной странице логически. Для твоего случая Leaf Bundle — более подходящий вариант.
Проверь с hugo server, и если нужно что-то доработать (например, обработку изображений в шаблоне), дай знать!
