# Как скрыть содержимое сайта если включён блокировщик рекламы.
Чтобы скрыть содержимое сайта при включённом блокировщике рекламы (например, AdBlock), можно использовать JavaScript для проверки наличия рекламы на странице. Если реклама заблокирована, вы можете скрыть основной контент и показать сообщение с просьбой отключить блокировщик. Вот один из способов реализации:
Пример кода
```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Сайт с защитой от AdBlock</title>
    <style>
        #content {
            display: none; /* Скрываем контент по умолчанию */
        }
        #adblock-message {
            display: none; /* Скрываем сообщение по умолчанию */
            color: red;
            font-size: 18px;
            text-align: center;
            margin-top: 20px;
        }
    </style>
</head>
<body>
    <!-- Тестовый рекламный блок -->
    <div id="ad" style="height: 1px; width: 1px;"></div>

    <!-- Основное содержимое сайта -->
    <div id="content">
        <h1>Добро пожаловать на сайт!</h1>
        <p>Здесь находится основной контент сайта.</p>
    </div>

    <!-- Сообщение для пользователей с AdBlock -->
    <div id="adblock-message">
        Пожалуйста, отключите блокировщик рекламы, чтобы увидеть содержимое сайта.
    </div>

    <script>
        // Функция проверки наличия рекламы
        function checkAdBlock() {
            var ad = document.getElementById('ad');
            // Если реклама заблокирована, высота или ширина будут нулевыми или элемент не отобразится
            if (!ad || ad.offsetHeight === 0 || ad.offsetWidth === 0) {
                document.getElementById('content').style.display = 'none';
                document.getElementById('adblock-message').style.display = 'block';
            } else {
                document.getElementById('content').style.display = 'block';
                document.getElementById('adblock-message').style.display = 'none';
            }
        }

        // Вызов функции после загрузки страницы
        window.onload = function() {
            checkAdBlock();
        };
    </script>

    <!-- Подключение "фейковой" рекламы для теста (обычно это настоящий рекламный скрипт) -->
    <script async src="//example.com/fake-ad.js"></script>
</body>
</html>
```
Как это работает:
Рекламный блок:
Создаётся элемент
```html
<div id="ad">
```
который имитирует рекламу. Обычно сюда подключается настоящий рекламный скрипт (например, Google AdSense).

Проверка: JavaScript проверяет, отображается ли этот блок. Блокировщики рекламы часто удаляют или скрывают такие элементы, что делает их размеры (offsetHeight или offsetWidth) равными нулю.

Реакция: Если реклама не обнаружена, основной контент (#content) остаётся скрытым, а показывается сообщение (#adblock-message).
Улучшения:
Обфускация кода: Чтобы затруднить обход защиты, можно обфусцировать JavaScript (например, с помощью инструментов вроде javascript-obfuscator).

Динамическая проверка: Используйте setInterval для периодической проверки, если пользователь включит AdBlock позже:
javascript
setInterval(checkAdBlock, 1000); // Проверка каждую секунду
Серверная проверка: Для большей надёжности можно комбинировать с серверной логикой, отправляя запросы и проверяя, загружаются ли рекламные ресурсы.
Замечания:
Некоторые продвинутые пользователи могут обойти такую защиту через консоль браузера или пользовательские скрипты.

Важно учитывать законы и правила вашей страны относительно таких методов, а также не раздражать пользователей чрезмерно.
Если нужны дополнительные детали или помощь с реализацией, дайте знать!
Без js
Скрыть содержимое сайта без использования JavaScript, если включён блокировщик рекламы, сложнее, так как большинство методов зависят от динамической проверки через скрипты. Однако есть обходной способ с использованием CSS и особенностей работы блокировщиков рекламы. Вот пример:
Пример кода без JavaScript
```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Сайт без JS с защитой от AdBlock</title>
    <style>
        /* По умолчанию скрываем контент */
        #content {
            display: none;
        }

        /* Сообщение для пользователей с AdBlock */
        #adblock-message {
            display: block;
            color: red;
            font-size: 18px;
            text-align: center;
            margin-top: 20px;
        }

        /* Если рекламный блок загружен, показываем контент и скрываем сообщение */
        #ad:not(:empty) ~ #content {
            display: block;
        }

        #ad:not(:empty) ~ #adblock-message {
            display: none;
        }

        /* Скрываем сам рекламный блок */
        #ad {
            height: 0;
            width: 0;
            overflow: hidden;
        }
    </style>
</head>
<body>
    <!-- Рекламный блок (имитация загрузки рекламы) -->
    <div id="ad">
        <!-- Обычно здесь подключается рекламный iframe или изображение -->
        <iframe src="//example.com/fake-ad.html" style="display:none;"></iframe>
    </div>

    <!-- Основное содержимое сайта -->
    <div id="content">
        <h1>Добро пожаловать на сайт!</h1>
        <p>Здесь находится основной контент сайта.</p>
    </div>

    <!-- Сообщение для пользователей с AdBlock -->
    <div id="adblock-message">
        Пожалуйста, отключите блокировщик рекламы, чтобы увидеть содержимое сайта.
    </div>
</body>
</html>
```
Как это работает:
CSS-селектор :not(:empty): Используется для проверки, пуст ли рекламный блок #ad. Если реклама загружается (блок не пуст), то применяются стили, показывающие контент и скрывающие сообщение.

Блокировщик рекламы: AdBlock часто блокирует загрузку ресурсов (например, <iframe> или <img> с доменов, связанных с рекламой). Если реклама заблокирована, #ad остаётся пустым, и контент не отображается.

Скрытие рекламы: Блок #ad имеет нулевые размеры, чтобы не мешать дизайну.
Ограничения:
Зависимость от блокировщика: Этот метод работает только если блокировщик полностью предотвращает загрузку рекламного контента. Некоторые блокировщики могут просто скрывать элемент, оставляя его в DOM, что сломает логику.

Тестирование: Нужно использовать реальный рекламный URL (например, от Google AdSense), так как блокировщики распознают их по сигнатурам. Подставной URL вроде fake-ad.html может не сработать.

Обход: Пользователи с техническими навыками могут легко обойти это через инспектор браузера, добавив содержимое в #ad вручную.
Альтернатива:
Если CSS-метод недостаточно надёжен, без JavaScript остаётся только серверная логика:
На сервере проверяйте, загружаются ли рекламные ресурсы (например, через запросы к рекламным сетям).

Если запросы не доходят (блокировщик их перехватил), отдавайте страницу с сообщением вместо контента.
Без JavaScript это менее гибкий подход, и я рекомендую всё же рассмотреть хотя бы минимальный JS для надёжности. Если есть вопросы или нужно адаптировать под конкретный случай, пишите!
Адоптируй метод к сайту на hugo
Hugo — это статический генератор сайтов, который не имеет встроенной серверной логики или динамического JavaScript по умолчанию. Однако мы можем адаптировать метод без JavaScript, используя CSS, к структуре Hugo. Поскольку Hugo работает с шаблонами и частичными файлами, я покажу, как встроить защиту от AdBlock в базовый шаблон.
Шаги для адаптации к Hugo
1. Создайте базовый шаблон
В Hugo шаблоны обычно находятся в папке layouts. Предположим, у вас есть базовый шаблон layouts/_default/baseof.html. Добавим в него логику защиты от AdBlock.
```html
<!-- layouts/_default/baseof.html -->
<!DOCTYPE html>
<html lang="{{ .Site.LanguageCode | default "ru" }}">
<head>
    <meta charset="UTF-8">
    <title>{{ .Title }}</title>
    <style>
        /* По умолчанию скрываем контент */
        .content {
            display: none;
        }

        /* Сообщение для пользователей с AdBlock */
        #adblock-message {
            display: block;
            color: red;
            font-size: 18px;
            text-align: center;
            margin-top: 20px;
        }

        /* Если рекламный блок загружен, показываем контент и скрываем сообщение */
        #ad:not(:empty) ~ .content {
            display: block;
        }

        #ad:not(:empty) ~ #adblock-message {
            display: none;
        }

        /* Скрываем рекламный блок */
        #ad {
            height: 0;
            width: 0;
            overflow: hidden;
        }
    </style>
</head>
<body>
    <!-- Рекламный блок -->
    <div id="ad">
        <!-- Используем реальный рекламный URL, например, Google AdSense -->
        <iframe src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js" style="display:none;"></iframe>
    </div>

    <!-- Основное содержимое -->
    <div class="content">
        {{ block "main" . }}{{ end }}
    </div>

    <!-- Сообщение для пользователей с AdBlock -->
    <div id="adblock-message">
        Пожалуйста, отключите блокировщик рекламы, чтобы увидеть содержимое сайта.
    </div>
</body>
</html>
```
2. Настройте шаблон для страниц
Создайте или отредактируйте файл layouts/_default/single.html (или другой шаблон, например, list.html), чтобы определить содержимое блока main. Например:
```html
<!-- layouts/_default/single.html -->
{{ define "main" }}
    <h1>{{ .Title }}</h1>
    {{ .Content }}
{{ end }}
```
3. Проверьте структуру сайта
Убедитесь, что ваш контент обёрнут в <div class="content"> через базовый шаблон.

Рекламный блок #ad должен содержать URL, который действительно распознаётся блокировщиками рекламы (например, от Google AdSense). Подставной URL вроде fake-ad.html не сработает, так как AdBlock его не заблокирует.
4. Сборка и тестирование
Запустите hugo для генерации сайта:
bash
hugo
Разместите сгенерированный сайт в папке public на локальном сервере (например, с помощью hugo server) и протестируйте:
bash
hugo server
Включите AdBlock в браузере и проверьте, скрывается ли .content и появляется ли #adblock-message.
Особенности для Hugo
Статичность: Поскольку Hugo генерирует статические HTML-файлы, метод полагается только на CSS и структуру DOM. Никакой динамической обработки на сервере или клиенте не требуется.

Рекламный URL: Используйте реальный рекламный скрипт или iframe (например, adsbygoogle.js от Google). Блокировщики рекламы фильтруют такие URL, оставляя #ad пустым, что активирует CSS-логику.

Кастомизация стилей: Вы можете перенести стили в отдельный файл CSS (например, static/css/style.css) и подключить его в <head>:
```html
<link rel="stylesheet" href="{{ "css/style.css" | relURL }}">
```
Пример структуры проекта
your-hugo-site/
├── layouts/
│   ├── _default/
│   │   ├── baseof.html    # Базовый шаблон с защитой
│   │   └── single.html    # Шаблон для страниц
├── static/
│   └── css/
│       └── style.css      # (опционально) Стили
├── content/
│   └── post/
│       └── example.md     # Контент
└── config.toml            # Конфигурация сайта

Ограничения
Как и в предыдущем методе без JS, это решение уязвимо к обходу через инспектор браузера.

Эффективность зависит от того, как блокировщик рекламы обрабатывает рекламный URL. Если он просто скрывает элемент, а не блокирует загрузку, метод может не сработать.
Если вам нужно что-то уточнить или добавить (например, поддержку многоязычности в Hugo), дайте знать!
Как это использовать с hugo pipeline
В Hugo есть мощная система обработки ресурсов (Asset Pipeline), которая позволяет управлять стилями, скриптами и другими файлами через встроенные функции, такие как resources.Get, resources.PostCSS, resources.Minify и т.д. Мы можем адаптировать предыдущий метод к Hugo Pipeline, чтобы стили были обработаны через эту систему, а структура оставалась чистой и модульной. Поскольку вы просили метод без JavaScript, я сосредоточусь на интеграции CSS и HTML-шаблонов с pipeline.
Шаги для интеграции с Hugo Pipeline
1. Подготовьте структуру проекта
Предположим, у вас стандартная структура Hugo:
your-hugo-site/
├── assets/
│   └── css/
│       └── main.css       # Файл со стилями
├── layouts/
│   ├── _default/
│   │   ├── baseof.html    # Базовый шаблон
│   │   └── single.html    # Шаблон для страниц
├── content/
│   └── post/
│       └── example.md     # Контент
└── config.toml            # Конфигурация сайта

2. Создайте стили в assets/css/main.css
Поместите CSS-логику для защиты от AdBlock в файл assets/css/main.css:
```css
/* assets/css/main.css */

/* По умолчанию скрываем контент */
.content {
    display: none;
}

/* Сообщение для пользователей с AdBlock */
#adblock-message {
    display: block;
    color: red;
    font-size: 18px;
    text-align: center;
    margin-top: 20px;
}

/* Если рекламный блок загружен, показываем контент и скрываем сообщение */
#ad:not(:empty) ~ .content {
    display: block;
}

#ad:not(:empty) ~ #adblock-message {
    display: none;
}

/* Скрываем рекламный блок */
#ad {
    height: 0;
    width: 0;
    overflow: hidden;
}
```
3. Обновите базовый шаблон baseof.html
Используйте Hugo Pipeline для подключения и обработки стилей:
```html
<!-- layouts/_default/baseof.html -->
<!DOCTYPE html>
<html lang="{{ .Site.LanguageCode | default "ru" }}">
<head>
    <meta charset="UTF-8">
    <title>{{ .Title }}</title>
    {{ $style := resources.Get "css/main.css" | minify | fingerprint }}
    <link rel="stylesheet" href="{{ $style.RelPermalink }}" integrity="{{ $style.Data.Integrity }}">
</head>
<body>
    <!-- Рекламный блок -->
    <div id="ad">
        <!-- Реальный рекламный URL, например, Google AdSense -->
        <iframe src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js" style="display:none;"></iframe>
    </div>

    <!-- Основное содержимое -->
    <div class="content">
        {{ block "main" . }}{{ end }}
    </div>

    <!-- Сообщение для пользователей с AdBlock -->
    <div id="adblock-message">
        Пожалуйста, отключите блокировщик рекламы, чтобы увидеть содержимое сайта.
    </div>
</body>
</html>
```
Объяснение Pipeline:
resources.Get "css/main.css": Берет файл стилей из папки assets/css.

minify: Минифицирует CSS для уменьшения размера.

fingerprint: Добавляет хэш к имени файла (например, main.123abc.css) и атрибут integrity для повышения безопасности и кэширования.

RelPermalink: Генерирует относительный URL для стилей.

integrity: Используется для проверки целостности файла (полезно для продакшена).
4. Настройте шаблон для страниц
Файл layouts/_default/single.html остаётся простым:
```html
<!-- layouts/_default/single.html -->
{{ define "main" }}
    <h1>{{ .Title }}</h1>
    {{ .Content }}
{{ end }}
```
5. Конфигурация (опционально)
Убедитесь, что в config.toml включена поддержка обработки ресурсов (обычно это включено по умолчанию):
```toml
# config.toml
baseURL = "http://example.com/"
languageCode = "ru"
title = "Мой сайт"

[build]
  writeStats = true  # Для оптимизации сборки
```
6. Сборка и тестирование
Сгенерируйте сайт:
```bash
hugo
```
Запустите локальный сервер для тестирования:
```bash
hugo server
```
Проверьте работу с включённым и выключенным AdBlock.
Преимущества использования Pipeline
Оптимизация: Стили автоматически минифицируются и кэшируются.

Безопасность: Fingerprinting добавляет контроль целостности.

Модульность: Вы можете легко добавлять новые стили в main.css или разбивать их на несколько файлов, объединяя через @import или resources.Concat.
Дополнительные улучшения
Несколько стилей: Если у вас много CSS-файлов, объедините их:
```html
{{ $style1 := resources.Get "css/main.css" }}
{{ $style2 := resources.Get "css/theme.css" }}
{{ $combined := slice $style1 $style2 | resources.Concat "css/bundle.css" | minify | fingerprint }}
<link rel="stylesheet" href="{{ $combined.RelPermalink }}" integrity="{{ $combined.Data.Integrity }}">
```
PostCSS: Если вы хотите использовать современные CSS-функции (например, autoprefixer), добавьте resources.PostCSS:
```html
{{ $style := resources.Get "css/main.css" | resources.PostCSS | minify | fingerprint }}
```
Ограничения
Как и раньше, метод без JS зависит от поведения блокировщика рекламы. Реальный рекламный URL (например, от AdSense) обязателен.

Hugo Pipeline работает только на этапе сборки, поэтому динамические проверки невозможны без добавления JavaScript.

Или использовать [blockadblock](https://blockadblock.com/)
